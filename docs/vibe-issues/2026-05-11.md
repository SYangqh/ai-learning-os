## 根因分析

### 侧边栏 done vs completed 不匹配

后端 `Stage.java` 中状态值为 `locked / active / completed`，但前端侧边栏写的是 `done`，导致已完成阶段渲染成 locked 样式（透明灰色不可点）。这类前后端枚举值不同步问题建议在 ARCHITECTURE.md 或 API 接口文档里明确列出状态枚举。

### 重复提交 artifact

`submitArtifact()` 提交后设置 `artifactStatus = 'submitted'`，但禁用条件只判断了 `passed`，漏掉了 `submitted` 状态。

### 评审通过动画时机

`loadArtifacts()` 在每次 `sendInput()` 后刷新，当 `artifactStatus` 从 `submitted` → `passed` 变化时，用 `useRef` 记录前一个值并触发庆祝动画。
