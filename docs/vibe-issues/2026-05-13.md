### D1-01 | 登录页/验证页使用硬编码 emerald 暗色主题，与三主题系统不兼容

**现象**：三主题系统切换后，learn 页面随主题变化，但 `page.tsx`（登录向导）和 `auth/verify/page.tsx` 仍然硬编码 `bg-gray-900`、`text-emerald-400`、`border-gray-700` 等暗色 Tailwind 类，在"正常"和"轻松"主题下背景色与面板色冲突。

**根因**：登录页和学习页是在不同阶段各自独立开发的，登录页照搬了早期暗色设计稿，没有经过主题系统统一。

**修复**：
- 登录页 `page.tsx` 全量替换为 `.t-*` 主题工具类（背景、面板、输入框、按钮、文字）
- `auth/verify/page.tsx` 同步替换
- 根容器添加 `t-bg` 防止背景色漏出

**防范规则**：每次新增页面，必须使用 `.t-*` 主题类，禁止直接写 `bg-gray-xxx`、`text-emerald-xxx` 等品牌色 Tailwind 类。

### D1-02 | 三主题切换器在 layout.tsx 未接入 ThemeProvider 时无法生效

**现象**：ThemeProvider 组件创建了但忘记在 `layout.tsx` 中注入，导致所有页面上的 `useTheme()` 返回默认值 `normal`，`data-theme` 属性从未被设置，CSS 变量失效。

**根因**：`layout.tsx` 依然使用硬编码的 `bg-gray-950 text-gray-100`，既覆盖了 CSS 变量，又没有 Provider 包裹。

**修复**：
- `layout.tsx` 移除 `bg-gray-950 text-gray-100`
- 添加 `import { ThemeProvider }` 并包裹 `children`
- body 改用 CSS 变量 `style={{ backgroundColor: 'var(--c-bg)', color: 'var(--c-text)' }}`

**防范规则**：凡是新增的 Context Provider，必须同步更新 `layout.tsx`，并在 SmokeTest 中验证 `<html data-theme>` 属性存在。

### D2-01 | 代码提交区 hardcoded，不适合非编程学科

**现象**：`learn/page.tsx` 中的代码编辑区（右侧面板）始终存在，对于学英文、学数学等非编程类学科没有意义，用户会困惑"为什么有一个空代码框"。

**根因**：代码区的显示/隐藏逻辑只依赖 `showCodePanel` 状态，没有读取 Skill YAML 中的 `artifact_type`，硬假设所有任务都是 CODE 类型。

**影响**：当框架扩展到英语、数学、政治等学科后，每个 Stage 的 TASK 节点都会强制展示代码区。

**现状处理**：本次不写代码，在 ARCHITECTURE.md 的 P1 阶段任务和新增 P3 章节中记录了完整设计：
- `artifact_type` 枚举（CODE / NOTE / QUIZ / DIAGRAM / NONE）
- 前端按 `artifact_type` 动态渲染产出区的改造路径
- 后端 Rubric 评审按类型分支的规划

**待做（P1 任务）**：
1. 后端 `SessionService` 在 `/session/start` 响应中返回 `artifact_type`
2. 前端读取 `artifact_type` 决定渲染代码区 / 文本区 / 选项卡 / 不显示
3. Skill YAML 补充 `artifact_type` 字段

### D3-01 | PowerShell here-string 中反引号被解析为 BEL 字符

**现象**：在 PowerShell here-string（`@" ... "@`）中尝试用双反引号 ` `` ` 转义 Markdown 代码块标记，结果字符 `` ` `` 被解析为 `\x07`（BEL 控制字符），写入文件后出现乱码。

**根因**：PowerShell here-string 中 `` `` `` 是转义序列，表示字面反引号，但在写入 UTF-8 文件时 PS 内部处理导致产生 `\x07`。

**修复**：改用 Python 脚本直接以 UTF-8 编码读写文件，绕过 PowerShell 转义问题。

**防范规则**：凡是需要在 PowerShell 中写入含反引号的 Markdown 文件，必须使用 Python 脚本（`open(..., encoding='utf-8')`），不能依赖 PowerShell here-string。
