## 问题清单

### P0 — 阻断编译的编码问题

#### 1. DynamicChatService.java BOM 字符导致级联编译失败

**现象**：`mvnw compile` 报 `非法字符: '\ufeff'` + 大量 `未结束的字符串文字`，同时 Lombok 注解处理失败，导致所有依赖 `@Slf4j`/`@Getter` 的文件报 `找不到符号 log`/`getId()` 等。

**根因**：上一轮会话中使用 PowerShell `Set-Content` 写入文件时采用了 UTF-8 with BOM 编码（Windows PowerShell 5.1 默认行为），Java 编译器不识别 BOM。

**修复**：用 `[System.IO.File]::WriteAllText(path, content, New-Object System.Text.UTF8Encoding $false)` 重写文件（显式指定无 BOM）。

**教训 → 规则**：凡用 PowerShell 写 Java/YAML/JSON 文件，必须用 `New-Object System.Text.UTF8Encoding $false`，绝不用 `Set-Content` 或 `Out-File` 默认模式。

---

#### 2. UserRateLimitInterceptor.java 双类定义 + 乱码中文字符串

**现象**：编译报 `类重复: UserRateLimitInterceptor`；截断旧类之后又报 `未结束的注释` / `未结束的字符串文字`。

**根因**：上一轮会话 `replace_string_in_file` 追加了新类定义到旧类定义之后，而不是替换。旧类注释和中文字符串均为 GBK 乱码（由 BOM 写入时的编码混乱导致）。

**修复**：同样用无 BOM UTF-8 完整重写文件，移除重复类。

### P1 — 设计问题

#### 3. Artifact 实体缺少 score/feedback 字段

**现象**：`ReportService` 初稿引用了 `artifact.getScore()` / `artifact.getFeedback()`，编译失败。

**根因**：Artifact 实体在 Phase 2 设计时只有 `status`，rubric 评审结果存放在 `LearningSession.progress` JSONB 里（`rubric_score`/`rubric_feedback`），没有冗余到 Artifact 上。

**修复**：ReportService 改为从 `session.getProgress()` 读取 rubric 结果，Artifact 只展示 `type` + `status` + `content` 预览。

**教训**：在引用实体字段前先用 `grep_search` 确认字段是否存在，不要凭记忆猜测。

---

#### 4. LearningSessionRepository 缺少 findByStageId

**现象**：ReportService 调用 `sessionRepository.findByStageId(stageId)`，Repository 接口未定义该方法。

**修复**：在 `LearningSessionRepository` 中添加 `List<LearningSession> findByStageIdOrderByStartedAtDesc(UUID stageId)`。
