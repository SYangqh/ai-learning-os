## Bug 2：AiResponseParser 降级链提前中断

### 现象

在编写 `AiResponseParserTest.parseJsonWithLessThan2Answers_FallbackToPlainText` 测试时发现：

当 AI 返回的 `suggestedAnswers` 数组少于 2 个时，`AiResponseParser.parseJsonNode()` 错误地返回 `ParseResult.plainText(question)`，导致降级链提前中断。

**预期行为**：应返回 `null`，让调用方继续尝试 OPTIONS 格式和纯文本解析。

**实际行为**：直接返回 `ParseResult.plainText(question)`，跳过后续降级尝试。

### 根因

`AiResponseParser.java` 第 127 行左右：

```java
if (suggestedAnswers.size() < 2) {
    log.warn("[AiResponseParser] 答案选项数量不足（<2），不显示预制答案");
    return ParseResult.plainText(question);  // ❌ 错误：提前返回
}
```

正确逻辑应该是：

```java
if (suggestedAnswers.size() < 2) {
    log.warn("[AiResponseParser] 答案选项数量不足（<2），不显示预制答案");
    return null;  // ✅ 允许调用方继续降级
}
```

### 修复方案

修改 `AiResponseParser.java` 第 ~127 行：

```java
return null;  // 替换原来的 return ParseResult.plainText(question);
```

### 影响范围

- ✅ 修复核心解析逻辑 bug
- ✅ 测试确认降级链正常运行
- ✅ 无需数据库迁移
- ✅ 无需前端改动

- [x] 其他 17 个 AiResponseParserTest 测试全部通过
