# Vibe Issues — 2026-05-12

## 问题清单

---

### P1-01 | `awaitsArtifact` 警告在 practice 节点误触发

**现象**：用户在 `practice` 节点回答了口头练习题，消息之后出现 "⚠ 此节点需要提交代码作品" 警告，用户误以为回答不合格。

**根因**：`practice → task` 推进后，后端返回 `awaits_artifact=true`（下一节点 `task` 需要代码），前端直接将该 flag 渲染为警告，时机上像"当前动作不合格"，实际是"下一步操作提示"。

**修复**：前端不再依赖后端 flag，改为从当前状态推导：
```ts
const awaitsArtifact = ARTIFACT_REQUIRED_NODES.has(currentNode) && artifactStatus === 'none'
```
只在用户已进入 `task` 节点且尚未提交时才显示提示，并调整措辞为"下一步需要提交代码作品"。

**防范规则**：节点提示文案要从"当前状态"推导，不要直接透传 server flag——server 说的是"next node needs artifact"，而不是"current reply is wrong"。

### P2-01 | 缺少"重新生成"功能

**现象**：用户收到一条不满意的 AI 回复，没有办法重新获取另一个版本，只能手动重新输入同样内容。

**修复**：
- 后端：新增 `POST /api/session/regenerateLast`——删除最后一条 assistant 消息，用剩余历史重新调用 LLM，节点状态不变
- 前端：最后一条 AI 消息气泡下方显示 `↺ 重新生成` 按钮，原地替换消息，不增加对话轮次

### P1-02 | 提交代码后 AI 评审响应慢，没有即时反馈

**现象**：用户点击「提交作品」后，需要再发消息触发 LLM 评审，整个过程无即时反馈，体感等待时间长。

**根因**：`submitArtifact()` 只调用了快速的 REST API（无 LLM），但没有在 UI 上给出任何文字确认，用户不知道系统已收到代码；再次发消息触发 LLM 评审时 LLM 响应本身耗时。

**修复**：`submitArtifact()` 成功后立即在聊天记录中追加一条本地 assistant 消息（不调用 LLM）：
> "✅ 代码已收到！请发送一条消息（如："开始评审"），AI 将立刻对你的代码进行 Rubric 评审。"

**防范规则**：涉及异步等待的操作，必须在发起端立即给出本地确认消息，不能让用户面对静默等待。

### P1-03 | 完成的阶段重新进入看不到历史聊天记录

**现象**：点击侧边栏中已完成（`completed`）的阶段，进入后是空白聊天，之前的对话全部消失。

**根因**：`PathService.startStage()` 只查找 `finishedAt IS NULL` 的活跃会话。已完成的阶段会话 `finishedAt != null`，查询返回 empty → 触发新建会话流程 → 调用 LLM 生成新开场白，历史被丢弃。

**修复**：在无活跃会话时，对 `completed` 状态的 stage 额外查询最近一次已完成会话（`findTopByStageIdAndUserIdOrderByStartedAtDesc`），直接返回其历史消息，不重新生成开场白。前端已有 `stageComplete` 逻辑，返回 `current_node=complete` 后会自动隐藏输入区，以只读模式展示。

**影响文件**：`LearningSessionRepository.java`（新增查询方法）、`PathService.java`（startStage 补充 completed 分支）

**防范规则**：查询"最新会话"时，必须分别考虑进行中（finishedAt IS NULL）和已完成（finishedAt IS NOT NULL）两种情况，不能假设阶段只有一种状态。

### P2-02 | 缺少当前节点操作提示

**现象**：用户不清楚每个节点（intro/concept/practice/task/review/retro）需要做什么操作，容易在练习节点写代码或在任务节点只口头回答。

**修复**：在输入区添加 `NODE_HINTS` 静态映射，根据 `currentNode` 显示一行蓝色提示文字，告知当前节点的预期操作（如"✏️ 口头回答练习题即可，无需写代码"）。

### P1-04 | 评审后右侧进度条不更新，一直显示"✓ 已提交 · 等待评审"

**现象**：用户发送消息触发 REVIEW 节点评审，AI 给出评审回复后，右侧代码面板的进度条/状态徽章仍显示"✓ 已提交 · 等待评审"，没有更新为"通过"或"需修改"。

**双重根因**：
1. `sendInput()` 缺少 `setRubricResult` 调用——`rubric_passed/score/feedback/hints` 字段从 advance 响应中读取但从未写入 React 状态，rubric 卡片不展示，且 `artifactStatus` 只依赖 `loadArtifacts`；
2. 评审通过后 `currentNode` 变为 `retro`，进度条的渲染条件 `currentNode === 'review'` 变 false——整个进度条消失，用户看不到最终结果；即便 `loadArtifacts` 正确更新了 `artifactStatus`，用户也因进度条消失而误以为状态没有更新。

**修复**：
- `sendInput()` 中在 `setNodeStatus` 后增加：若 `d.rubric_passed !== undefined`，立即 `setRubricResult(...)` + `setArtifactStatus(passed → 'passed' / failed → 'needs_revision')`，不再依赖 `loadArtifacts` 的异步延迟；
- 进度条渲染条件从 `currentNode === 'review' && artifactStatus !== 'none'` 改为 `artifactStatus !== 'none' && !stageComplete`——只要有提交记录就持续显示进度条，阶段完成后隐藏。

**防范规则**：advance 响应中的额外字段（如 rubric_*）必须在 `sendInput` 中显式处理，不能只在 `openStage`（启动时恢复）处理；UI 状态应从最快的数据源（response 直接返回）设置，`loadArtifacts` 作为兜底刷新而非唯一来源。
