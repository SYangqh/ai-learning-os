## 新功能规划

### 📋 用户反馈与日志采集系统（Phase 11 候选）

**背景**：
产品上线后需要用户能够直接反馈遇到的问题，同时系统需要自动采集日志以便快速定位和复现问题。

**核心需求**：
1. **前端反馈入口**：用户可以在任意页面提交反馈（文字描述 + 可选截图）
2. **前端日志采集**：自动采集用户操作轨迹、前端错误、API 调用失败等
3. **后端日志关联**：反馈提交时自动附带用户最近 N 分钟的前端日志和相关后端 trace_id
4. **敏感信息脱敏**：自动过滤 API Key、token、密码等敏感字段
5. **管理后台（后续）**：查看反馈列表、关联日志、标记处理状态

**设计文档位置**：
- 详见 `docs/USER_FEEDBACK_AND_LOGGING_DESIGN.md`（已创建，共 11 章节，约 600 行）

**核心设计要点**：
1. **前端日志采集**: FrontendLogger 服务，IndexedDB 本地缓存（30 分钟滚动窗口，最多 500 条）
2. **反馈入口**: FeedbackButton 全局悬浮按钮 + FeedbackDialog 表单弹窗
3. **敏感信息脱敏**: 前后端双重脱敏（API Key/Token/邮箱等）
4. **数据库设计**: user_feedbacks + frontend_event_logs 表（V8 Flyway）
5. **管理后台**: 反馈列表/详情/状态管理/日志查看/导出功能
6. **存储策略**: 前端 IndexedDB 自动清理；后端按月分区，3 个月后归档

**实施优先级**：
- Phase 11 或更晚（当前 Phase 9B 完成后，Phase 9C/10A/10B/10C 优先）
- 建议在 Phase 10 完成后、正式上线前实施
- 预计工期：9 个工作日（管理后台延后则 6 天）
