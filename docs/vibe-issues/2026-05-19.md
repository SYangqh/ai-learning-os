## Design Decision 1：动态预制答案扩展方案

### 背景

当前预制答案机制只覆盖 Skill YAML 中人工预定义的问题场景，AI 生成的深入追问无法提供预制答案选项，导致用户体验不一致。

### 方案对比

| 方案 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| A. YAML Only | 质量可控 | 覆盖率低，维护成本高 | ❌ 不采用 |
| B. AI Only | 覆盖率 100%，零维护 | 质量不稳定，无人工把关 | ❌ 不采用 |
| **C. Hybrid（YAML 优先 + AI 降级）** | 质量与覆盖率兼顾 | 实现复杂度中等 | ✅ **采用** |

### 方案 C 设计细节

#### 数据流

```
用户回答 → 后端加载 Skill YAML → 关键词匹配
                                ↓ 有匹配
                        返回 YAML 预制答案（source: YAML）
                                ↓ 无匹配
                        AI 生成结构化 JSON
                                ↓
                    {
                      question: "...",
                      suggestedAnswers: [...]
                    }
                                ↓
                返回 AI 动态生成预制答案（source: AI_GENERATED）
```

#### YAML 扩展字段

在 `preset_answers` 中新增 `trigger_keywords` 字段：

```yaml
preset_answers:
  - id: pa_nodejs_module_yes
    text: "用过 import/require，引入过第三方库"
    confidence: HIGH
    trigger_keywords: ["模块", "包", "import", "require", "CommonJS", "ES Module"]
    stage_index: null
```

#### AI Prompt 要求

AI 必须返回符合以下 JSON Schema 的结构化输出：

```json
{
  "question": "string (AI 生成的问题)",
  "suggestedAnswers": [
    {
      "text": "string (答案选项文本)",
      "confidence": "HIGH | MEDIUM | LOW"
    }
  ]
}
```

#### 优先级规则

1. **优先级 1**：YAML 全局预制答案（`stage_index: null`, `trigger_keywords: []`）
2. **优先级 2**：YAML 关键词触发预制答案（`trigger_keywords` 匹配）
3. **优先级 3**：YAML 阶段级预制答案（`stage_index` 匹配）
4. **优先级 4**：AI 动态生成预制答案（解析 JSON `suggestedAnswers`）

#### 降级策略

| 异常情况 | 处理方式 |
|---------|---------|
| AI 未返回 `suggestedAnswers` | 不显示预制答案面板 |
| AI 返回格式错误（非 JSON） | 解析失败时降级为纯文本问题 |
| AI 返回的答案选项 <2 个 | 不显示预制答案面板 |
| AI 返回的答案选项 >5 个 | 只取前 5 个 |
| 关键词匹配返回 >10 个预制答案 | 只取前 5 个（按 confidence 排序） |

### 实施计划

参见 `docs/PHASE_9D_IMPLEMENTATION_PLAN.md`，预计工作量 8-12 小时。

### 验收标准

- [ ] **场景 1（YAML 全局）**：初始问题显示 YAML 全局预制答案
- [ ] **场景 2（YAML 关键词）**：AI 追问"模块"时显示关键词触发的预制答案
- [ ] **场景 3（AI 动态生成）**：AI 追问"循环依赖"时显示 AI 动态生成的预制答案
- [ ] **场景 4（降级）**：AI 返回格式错误时不显示预制答案，允许自由输入

### 风险评估

1. **AI 不稳定返回 JSON**：通过健壮的降级策略缓解
2. **关键词匹配误触发**：通过人工测试迭代关键词库
3. **性能影响**：通过缓存和异步处理缓解

## Phase 9C：移动端 Web 与 iOS / Android 验证

### 问题 1：`loadDraftFromStage` 无法复用 hook 的 `loadDraft`

**背景**：`useOfflineDraft` 的 `loadDraft()` 绑定了 hook 初始化时的 `stageId`（通过 `activeStage?.id`）。但 `openStage` 函数在设置 `activeStage` 之前需要读取新 stageId 的草稿，导致 hook 还未绑定新 stageId。

**根因**：React 状态的异步特性 —— `setActiveStage(stage)` 调用后，`activeStage` 在当前 render cycle 内仍是旧值，hook 的 stageId 也是旧值。

**解决方案**：在 `learn/page.tsx` 内单独实现 `loadDraftFromStage(stageId, type)` 直接操作 IndexedDB，绕过 hook 的 stageId 绑定。这确保了在状态更新之前也能正确读取草稿。

**可操作规则**：禁止在状态尚未更新时依赖绑定了该状态的 hook 的衍生方法；需要直接读的场景应提取独立工具函数。

**技术债记录（待重构）**：`learn/page.tsx` 中的 `loadDraftFromStage` 复制了 `useOfflineDraft.ts` 内 `openDB` + IndexedDB 读取逻辑，形成维护重复。后续应将 `openDB`、`saveDraftToDB`、`loadDraftFromDB`、`clearDraftFromDB` 从 `useOfflineDraft.ts` 提取到 `src/lib/draftDB.ts`（纯工具模块，不含 React），让 hook 和 `loadDraftFromStage` 共享同一套实现。重构步骤：
1. 新建 `frontend/src/lib/draftDB.ts`，导出 `openDB / saveDraftToDB / loadDraftFromDB / clearDraftFromDB`
2. `useOfflineDraft.ts` 改为从 `@/lib/draftDB` import
3. `learn/page.tsx` 的 `loadDraftFromStage` 改为直接调用 `loadDraftFromDB(stageId, type)`，删除内联重复代码

### 技术决策记录

#### 决策 1：混合方案优先级

**决策**：YAML 优先 > AI 动态生成 > 无预制答案

**理由**：
- YAML 预制答案是人工精心编写的,质量更高
- AI 动态生成作为降级方案,保证全覆盖
- 避免 AI 不稳定性影响核心用户体验

**风险缓解**：
- 关键词匹配逻辑简单且快速（<10ms）
- AI JSON 解析失败时降级为纯文本,不影响核心流程

#### 决策 2：关键词匹配策略

**决策**：使用子串匹配（不区分大小写）,支持数组关键词

**理由**：
- 简单高效,易于维护
- 支持中文和英文关键词混合
- 避免复杂的 NLP 依赖

**潜在问题**：
- 可能误触发（如"这个模块我们先不讲"包含"模块"）
- 关键词过于宽泛会导致匹配不精确

**后续优化方向**：
- 可以增加"否定关键词"机制
- 可以升级为向量语义匹配

#### 决策 3：AI 结构化输出格式

**决策**：要求 AI 返回 JSON 格式,但支持降级为纯文本

**理由**：
- GPT-4 和 Claude 3.5 Sonnet 对结构化输出支持较好
- JSON 格式便于前后端解析
- 降级策略保证健壮性

**风险评估**：
- AI 可能不稳定返回 JSON（估计成功率 70-80%）
- 降级为纯文本时,用户体验会降级为无预制答案
- 但不影响核心对话流程

#### 决策 4：前端来源标记

**决策**：只在 AI 生成的预制答案显示"（AI 智能生成）"标记

**理由**：
- YAML 预制答案无需标记（用户默认认知）
- AI 生成的预制答案需要透明度,让用户知道来源
- 标记简洁,不干扰视觉
