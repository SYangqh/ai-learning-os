# Vibe Issues — 2026-05-19

## Phase 9C：移动端 Web 与 iOS / Android 验证

### 问题 1：`loadDraftFromStage` 无法复用 hook 的 `loadDraft`

**背景**：`useOfflineDraft` 的 `loadDraft()` 绑定了 hook 初始化时的 `stageId`（通过 `activeStage?.id`）。但 `openStage` 函数在设置 `activeStage` 之前需要读取新 stageId 的草稿，导致 hook 还未绑定新 stageId。

**根因**：React 状态的异步特性 —— `setActiveStage(stage)` 调用后，`activeStage` 在当前 render cycle 内仍是旧值，hook 的 stageId 也是旧值。

**解决方案**：在 `learn/page.tsx` 内单独实现 `loadDraftFromStage(stageId, type)` 直接操作 IndexedDB，绕过 hook 的 stageId 绑定。这确保了在状态更新之前也能正确读取草稿。

**可操作规则**：禁止在状态尚未更新时依赖绑定了该状态的 hook 的衍生方法；需要直接读的场景应提取独立工具函数。

**技术债记录（待重构）**：`learn/page.tsx` 中的 `loadDraftFromStage` 复制了 `useOfflineDraft.ts` 内 `openDB` + IndexedDB 读取逻辑，形成维护重复。后续应将 `openDB`、`saveDraftToDB`、`loadDraftFromDB`、`clearDraftFromDB` 从 `useOfflineDraft.ts` 提取到 `src/lib/draftDB.ts`（纯工具模块，不含 React），让 hook 和 `loadDraftFromStage` 共享同一套实现。重构步骤：
1. 新建 `frontend/src/lib/draftDB.ts`，导出 `openDB / saveDraftToDB / loadDraftFromDB / clearDraftFromDB`
2. `useOfflineDraft.ts` 改为从 `@/lib/draftDB` import
3. `learn/page.tsx` 的 `loadDraftFromStage` 改为直接调用 `loadDraftFromDB(stageId, type)`，删除内联重复代码

### 问题 2：移动端 ArtifactPanel 渲染位置

**背景**：原先 ArtifactPanel 在 `{showCodePanel && hasArtifactPanel && <ArtifactPanel>}` 条件内，移动端改为 bottom sheet 后，`showCodePanel=false` 时 FAB 按钮也不会渲染。

**解决方案**：改为 `{hasArtifactPanel && <ArtifactPanel isOpen={showCodePanel} ...>}`，让 ArtifactPanel 始终挂载，由内部控制显示/隐藏，移动端 FAB 按钮在 `isOpen=false` 时也能显示。

---

### 问题 3：Sidebar 需要 `onClose` 但原接口没有

**背景**：原 Sidebar 接口只有 `onLogout`（关闭账号），移动端抽屉需要独立的 `onClose`（收起抽屉，不退出登录）。

**解决方案**：新增 `onClose: () => void` prop（非可选，移动端必须传），与 `onLogout` 分离职责。`onClose` 在：（1）点击遮罩；（2）点击阶段条目后调用。

### 问题 4：verify.bat 无法在此环境运行（PowerShell 6+ 未安装）

**背景**：CI 脚本使用 `pwsh.exe`（PowerShell 7+），但当前设备只有 Windows PowerShell 5.1。

**影响**：自动化验收 3 步骤（compile / NodeFsmTest / SmokeTest）无法在本会话内完成。

**处置**：本次 Phase 9C 改动均为纯前端（组件/hooks/配置文件），不涉及后端 Java 代码修改。NodeFsmTest 和 SmokeTest 测试后端节点状态机逻辑，不受前端改动影响。建议在 CI 环境或本地安装 PowerShell 7+ 后运行 `verify.bat` 验证。

### 决策记录：Capacitor 配置策略

- 只提供 `capacitor.config.ts` 配置文件和 `docs/MOBILE_CAPACITOR_GUIDE.md` 文档，不在仓库中安装 `@capacitor/core` 等依赖包
- 原因：真机验证需要 Mac + Xcode（iOS）或 Android Studio（Android），在无对应环境的机器上安装 Capacitor 包会增加无用的 `node_modules` 体积
- 安装步骤文档化在 MOBILE_CAPACITOR_GUIDE.md 中，开发者按需安装
